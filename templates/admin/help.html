Getting started

Centralized is a self-hosted pentest management platform. It lets you organise your work around three main objects:

  • Clients — the companies or entities you are testing for.
  • Audits — one engagement per client (web app pentest, infra audit, etc.).
  • Hosts — every IP / machine discovered during an audit.

You import scan results from tools such as Nmap, Nuclei, Nikto, httpx, SQLMap, Lynis, or DirBuster, and Centralized parses them automatically into structured vulnerabilities, ports, and findings that you can review, score, and export to PDF.

Start by creating a Client, then an Audit for that client, then upload your scan files — everything else follows naturally.

On the very first launch Centralized creates a default administrator account:

Username : admin Password : admin
Change this password immediately — go to Administration → Users and click Edit on the admin account.

You can create additional accounts (admin or analyst roles) from the same Users page.

1Create a Client (Clients → New client).
2Create an Audit for that client (Audits → New audit).
3Run your scans externally (Nmap, Nuclei, Nikto…).
4Upload the result files to the audit.
5Review discovered Hosts and Vulnerabilities.
6Enrich CVEs, set statuses, add manual findings.
7Export the PDF report.
8Mark the audit as Completed.

SSL certificate & HTTPS

Centralized auto-generates a self-signed certificate at installation time. Browsers don't trust self-signed certs by default — hence the warning. The connection is still fully encrypted; you just need to tell your browser (or OS) to trust this specific certificate.

For production use you should replace the cert with one issued by Let's Encrypt or another CA.

Option A — Chrome / Edge (via Windows cert store)
  1. Navigate to the app URL and click Advanced → Proceed to reach the page.
  2. Click the 🔒 lock icon in the address bar → Certificate is not valid.
  3. Switch to the Details tab → Copy to file… and export the cert as DER .cer.
  4. Double-click the exported .cer file → Install Certificate.
  5. Choose Local Machine → Place all certificates in the following store → Trusted Root Certification Authorities.
  6. Click Finish and restart Chrome/Edge.
Option B — PowerShell (quick, one-liner)
Import-Certificate -FilePath "ssl\cert.pem" -CertStoreLocation Cert:\LocalMachine\Root

Run as Administrator. The cert file is in the Centralized install directory under ssl\cert.pem.

Keychain Access (GUI)
  1. Open Keychain Access (Spotlight → Keychain Access).
  2. Drag and drop ssl/cert.pem onto the login keychain.
  3. Double-click the imported cert → expand Trust → set When using this certificate to Always Trust.
  4. Close and authenticate with your password.
  5. Restart your browser.
Terminal (one-liner)
sudo security add-trusted-cert -d -r trustRoot -k /Library/Keychains/System.keychain ssl/cert.pem

Debian / Ubuntu
sudo cp ssl/cert.pem /usr/local/share/ca-certificates/centralized.crt sudo update-ca-certificates
RHEL / Fedora / CentOS
sudo cp ssl/cert.pem /etc/pki/ca-trust/source/anchors/centralized.crt sudo update-ca-trust

After running these commands, restart your browser.

Firefox uses its own certificate store independent of the OS.

  1. Open about:preferences#privacy.
  2. Scroll down to Certificates → click View Certificates.
  3. Switch to the Authorities tab → click Import….
  4. Select ssl/cert.pem.
  5. Check Trust this CA to identify websites → OK.

Clients & Audits

  1. In the sidebar, click Clients.
  2. Click the New client button (top right).
  3. Fill in:
    • Name (required) — e.g. Acme Corp.
    • Description — optional context about the client.
    • Contact — e-mail or name of the point of contact.
  4. Click Create client.
A client can have multiple audits. Think of a client as a company and an audit as a specific engagement (pentest, compliance review, etc.).

  1. In the sidebar, click Audits.
  2. Click New audit.
  3. Fill in:
    • Audit name — e.g. Web App Pentest – March 2026.
    • Client — select the associated client (optional).
    • Target — the IP range, domain, or scope description.
    • Scope — additional scope notes.
    • Start / end date — for the report header.
  4. Click Create audit.

Once created, the audit page gives you access to:

  • Discovered hosts & vulnerabilities
  • Upload scan files
  • Manual findings
  • PDF report export
  • Status management (In progress / Completed / Archived)

  • In progress — active engagement, data can be added freely.
  • Completed — engagement finished; fires a notification to subscribers.
  • Archived — closed for editing, kept for reference.

Change the status from the audit detail page using the status selector in the top-right card.

Importing scan data

Centralized auto-detects the following formats:

ToolFormatHow to generate
NmapXML (.xml)nmap -oX output.xml <target>
NmapJSON (.json)via nmap converter scripts
AutoReconJSON (.json)built-in output
httpxJSON (.json)httpx -json -o output.json
NucleiJSON (.json)nuclei -json -o output.json
NiktoXML or JSONnikto -Format xml -output out.xml
Lynislog / reportlynis audit system → /var/log/lynis.log
SQLMapTXT or CSVsqlmap --output-dir ./out
DirBuster / ffufJSON or TXTffuf -o output.json -of json
ScreenshotsPNG / JPGany screenshot tool
PDF reportsPDFany — stored as attachment
You can upload multiple files in one go — drag & drop them all at once into the upload zone.

  1. Open the audit detail page.
  2. Click Upload files (button in the top section or via the sidebar).
  3. Drag & drop your files onto the zone, or click Browse Files.
  4. For some file types (Lynis, SQLMap, DirBuster, screenshots) you will be prompted to enter the target IP so the parser can link data to the correct host.
  5. Click Upload.
  6. After upload, Centralized parses the files automatically. Hosts, ports, and vulnerabilities are created or merged with existing data.
Re-uploading the same Nmap scan won't create duplicate hosts — the parser merges on IP address.

nmap -sV -sC -O -p- --open -oX nmap_output.xml <target>

Flags breakdown:

  • -sV — service version detection
  • -sC — default scripts (banner grab, common checks)
  • -O — OS detection
  • -p- — all 65535 ports
  • --open — only show open ports
  • -oX — output as XML (required for Centralized import)
For faster scans on large ranges: use -T4 and limit to common ports first with --top-ports 1000, then re-run with -p- on confirmed live hosts.

nuclei -l targets.txt -severity critical,high,medium -json -o nuclei_output.json
  • -l targets.txt — list of URLs/IPs to scan
  • -severity — filter by severity
  • -json — JSON output (required)
  • -o — output file

Hosts & Vulnerabilities

Each host page shows:

  • Risk score — automatically computed from the severity and count of vulnerabilities.
  • Open ports — with service name, version, and banner.
  • Vulnerabilities — patched in from Nuclei / Nikto / manual input.
  • HTTP pages — web surfaces discovered via httpx.
  • CVE enrichment — click the ⚡ button on a vuln to pull CVSS, EPSS, and remediation from NVD.
  • Host context — manually set OS version, build number, and service versions for report accuracy.

  • Open — active, not yet remediated.
  • In progress — remediation underway.
  • Risk accepted — client has accepted the risk.
  • False positive — confirmed non-issue; excluded from risk score.
  • Patched — fix verified and applied.

Change the status directly in the vulnerability row — the risk score updates immediately.

  1. Open the host detail page.
  2. Scroll to the Vulnerabilities section.
  3. Click Add vulnerability.
  4. Fill in the name, CVE ID (optional), severity, description, and port.
  5. Save — it will immediately affect the host's risk score.

For any vulnerability with a CVE ID, Centralized can pull enrichment data from NVD (National Vulnerability Database):

  • CVSS v3 / v4 base score and vector
  • EPSS score (exploit probability)
  • Description
  • Remediation / patch advice
  • References

Click the Enrich button on a vulnerability row to fetch data for that single CVE, or use the Enrich all button at the top of the host to process all CVEs at once.

Set a GitHub API token in Administration → Settings to raise the NVD rate limit from 60 to 5 000 requests/hour.

CVE Search & Remediation

The CVE Search page (sidebar) lets you query the NVD database directly:

  • Search by CVE ID (e.g. CVE-2021-44228)
  • Search by keyword (e.g. Log4j remote code execution)
  • Filter by severity (Critical / High / Medium / Low)
  • Filter by date range

From the results you can click any CVE to see the full detail, CVSS score, and available patches.

The Remediation section gives you AI-assisted step-by-step remediation plans for any CVE. It queries configured CVE sources (NVD, CIRCL, MITRE, etc.) and consolidates the best remediation advice.

  1. Navigate to CVE Remediation in the sidebar.
  2. Enter a CVE ID.
  3. Select the impacted environment (OS, software version).
  4. Review the generated plan — steps are priority-ordered.
  5. Export to PDF if needed.

AutoRecon integration

AutoRecon is a multi-threaded reconnaissance tool that runs many scanning tools (Nmap, Nikto, ffuf, enum4linux-ng, etc.) against a target and organises all results into a structured directory.

Centralized can launch AutoRecon scans directly from the UI and then automatically import the results into an audit.

The AutoRecon section only appears in the sidebar when AutoRecon is installed on the system. Run the AutoRecon.sh / AutoRecon.bat installer first.

  1. Click AutoRecon in the sidebar.
  2. Enter the target (IP, range, or hostname).
  3. Select the audit to link results to.
  4. Optionally configure extra AutoRecon flags.
  5. Click Launch scan.
  6. Live terminal output streams in the browser window.
  7. When the scan finishes, parsed results (hosts, ports, vulns) are automatically imported into the selected audit.

Notifications

  1. Go to Notifications in the sidebar.
  2. Click Request permission — your browser will show a permission prompt.
  3. Click Allow.
  4. Make sure the Global notifications toggle is Enabled.
  5. For each client / audit / host you want to monitor, click the bell icon and choose which events trigger a notification.
Notifications are polled every 30 seconds. Keep the app tab open (or install as a PWA) so you receive them.

Per client
  • New audit created — a new audit was added for this client.
  • Audit completed — an audit for this client was marked as completed.
  • New critical vulnerability — a critical CVE was discovered in any audit of this client.
Per audit
  • Status change — the audit status changed.
  • New host discovered — a new host was added to the audit.
  • New critical vulnerability — same as above, audit-scoped.
  • Audit completed — the audit was marked as completed.
Per host
  • New vulnerability found — any new vuln added to this host.
  • Critical / High vulnerability — a critical or high severity vuln was added.
  • Risk score changed — the computed risk score changed.

Reports

  2. Click the Export PDF button.
  3. The report includes:
    • Executive summary

Interface & Settings

  1. Go to Administration → Interface.
  2. Use the color pickers to adjust: Accent color, Sidebar background, Card background, Body background.
  3. A live preview updates in real time on the right.
  4. Click Save theme.

You can also extract colors from an image: drop a wallpaper or screenshot into the drop zone at the top — Centralized will suggest a palette derived from the image's dominant colors.

The Glassmorphic toggle adds a frosted-glass effect across the whole UI.

  1. Go to Administration → Interface and scroll to the App Icon card.
  2. Click Choose file and select a PNG, JPG, WEBP, or GIF (min. 192×192 px recommended).
  3. A preview shows the selected image.
  4. Click Save icon.

The custom icon replaces the favicon in browser tabs, the home screen shortcut (PWA), and the Apple touch icon. Click Reset to default to revert.

Go to Administration → Settings to manage the CVE data sources Centralized queries for enrichment and search:

  • NVD — built-in, always active, official US government database.
  • CIRCL, MITRE, EPSS, OSV, ENISA EUVD, Tenable, Wiz, VulDB… can all be added.
  • Toggle sources on/off or add custom URLs.

Go to Administration → Users (admin only):

  • Create new users with role admin or analyst.
  • Edit username, e-mail, or password of any account.
  • Delete accounts (cannot delete your own).

Roles:

  • admin — full access including user management, interface settings, and update.
  • analyst — full access to clients, audits, hosts, CVE search; no access to admin pages.

Updates & maintenance

When an update is available, a pulsing badge appears next to Update in the sidebar.

  1. Click Update in the Administration section.
  2. Review the latest version info and changelog.
  3. Click Run update.
  4. The terminal output streams live. On Linux with systemd, the service restarts automatically.
  5. On Windows, restart the Centralized service manually via Services or the task manager.
The database schema is migrated automatically — no data is lost during a normal update.

You can also rollback to a previous version from the same page by selecting a commit from the version history.

Centralized stores all data in an SQLite database file and an uploads/ directory. To back up manually:

# Copy the database cp centralized.db centralized.db.bak # Copy uploads cp -r uploads/ uploads_backup/

On Windows:

Copy-Item "centralized.db" "centralized.db.bak" Copy-Item "uploads\" "uploads_backup\" -Recurse
Automate this with a cron job (Linux) or Task Scheduler (Windows) to keep daily backups.

Troubleshooting

  • Check that the file format is supported (see Supported file formats above).
  • For Nmap, make sure the file was generated with -oX (XML), not -oN or -oG.
  • For Nuclei, make sure you used -json, not plain text output.
  • Files with 0 bytes or corrupted content will show as uploaded but produce no results — re-run the tool and re-upload.
  • Check the audit's uploaded files list: the file type is shown next to each entry. Unknown means the parser could not identify the format.

  • Check that the browser permission is Granted on the Notifications page.
  • Make sure the global toggle is Enabled.
  • Make sure you have subscribed to at least one event for the relevant entity (bell icon).
  • Some OSes (Windows Focus Assist, macOS Do Not Disturb) can suppress browser notifications. Disable them temporarily to test.
  • Notifications are polled every 30 seconds — there may be a short delay after an event fires.

The NVD API limits anonymous requests to 60 per hour. With a GitHub PAT (no special scopes required) Centralized can make 5 000 requests per hour.

  1. Go to github.com/settings/tokens.
  2. Generate a classic token with no scopes (read-only public API access).
  3. Paste it in Administration → Settings → GitHub API Token.

Centralized tries to bind port 80 for the HTTP→HTTPS redirect. This fails if:

  • Another process (Apache, nginx, IIS…) is already using port 80.
  • The app doesn't have permission to bind privileged ports on Linux (ports < 1024 require root or CAP_NET_BIND_SERVICE).

To fix on Linux without running as root:

sudo setcap 'cap_net_bind_service=+ep' $(which python3)

Or use a reverse proxy (nginx / caddy) in front of Centralized.

  • PWA installation requires HTTPS — HTTP is not supported by browsers for PWA.
  • The SSL certificate must be trusted (not just bypassed) — see the SSL section above.
  • On Chrome, open chrome://apps and check if the site is already installed.
  • On Edge, click the 3-dot menu → Apps → Install this site as an app.
